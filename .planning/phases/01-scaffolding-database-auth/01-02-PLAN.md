  <action>
    **Create `src/lib/auth.ts`.**
    Use the exact pattern from 01-RESEARCH.md Pattern 3:
    - Add `import 'server-only'` at top (install server-only package if not present: `npm install server-only`)
    - `validatePassword(input: string): Promise<boolean>` — reads AUTH_PASSWORD env var, throws if not set. If stored starts with '$2' (bcrypt hash), use bcryptjs.compare(). Otherwise, direct string comparison.
    - `createSession(): Promise<void>` — generates 32-byte random token via crypto.randomBytes, calculates expiresAt as now + 30 days, inserts into sessions table via getDb(), sets httpOnly secure cookie named 'session' with 30-day maxAge. CRITICAL: cookies() is async in Next.js 15 — must await.
    - `deleteSession(): Promise<void>` — reads session cookie, deletes matching row from sessions table, deletes cookie. CRITICAL: await cookies().
    - `refreshSession(token: string): void` — updates expiresAt to now + 30 days for the given token (called from middleware to extend session on activity).

    **Create `src/app/api/auth/login/route.ts`.**
    - POST handler: reads JSON body `{ password }`, calls validatePassword(). If valid, calls createSession(), returns NextResponse.json({ success: true }). If invalid, returns 401 with { error: 'Invalid password' }.
    - If AUTH_PASSWORD is not set, return 500 with clear error.

    **Create `src/app/api/auth/logout/route.ts`.**
    - POST handler: calls deleteSession(), returns NextResponse.json({ success: true }).

    **Create `src/app/(auth)/login/page.tsx`.**
    - The (auth) route group means this page does NOT get the dashboard layout.
    - Client component ('use client') with a form containing password input and submit button.
    - On submit: POST to /api/auth/login with { password }. On success: redirect to '/' using router.push('/'). On failure: show error message below input.
    - Styled with shadcn/ui: Card centered on screen, Input for password, Button for submit.
    - Dark background matching dashboard theme. Show "Social Content Engine" title above the card.
    - Loading state on button during submission.
    - No "register" or "forgot password" links (single-user tool).
  </action>

  <action>
    **Replace the existing `middleware.ts`** (currently contains instrumentation-related code from Phase 0) with auth middleware.

    Use the pattern from 01-RESEARCH.md Pattern 2:
    - CRITICAL: Export `config` with `runtime: 'nodejs'` — this is stable in Next.js 15.5 and required for better-sqlite3 access.
    - Matcher: exclude `_next/static`, `_next/image`, `favicon.ico`, `login`, `api/health`, `api/auth` from middleware. Pattern: `'/((?!_next/static|_next/image|favicon.ico|login|api/health|api/auth).*)'`
    - In the middleware function:
      1. Read `session` cookie from request
      2. If no cookie: redirect to /login
      3. If cookie exists: call getDb(), query sessions table for matching token where expiresAt > now
      4. If no valid session: delete the cookie and redirect to /login
      5. If valid session: optionally refresh the session expiry (extend 30 days), return NextResponse.next()
    - Use drizzle-orm operators: eq, and, gt from 'drizzle-orm'
    - Import sessions from '@/db/schema'

    **IMPORTANT:** The existing `instrumentation.ts` file should remain as-is (it was created in Phase 0). The middleware.ts file at the project root is a separate file.

    **Note about imports:** In middleware with `runtime: 'nodejs'`, ES module imports work normally. Use standard imports for getDb and schema — no need for require().
  </action>
