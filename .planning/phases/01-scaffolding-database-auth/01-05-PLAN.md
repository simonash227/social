  <action>
    **Create `src/app/actions/accounts.ts`.**
    - Add 'use server' directive
    - Import getDb, socialAccounts schema, listProfiles from upload-post client

    **`syncAccounts(brandId: number): Promise<{ synced: number, error?: string }>`**
    - Call `listProfiles()` from the Upload-Post client (wrapped in try/catch)
    - If API call fails (circuit breaker open, network error): return { synced: 0, error: message }
    - For each profile returned:
      - For each platform in profile.connected_platforms:
        - Check if a socialAccount with brandId + platform + username already exists
        - If not: insert new socialAccount with status 'connected', failureCount 0
        - If exists and status is 'disconnected' but profile is active: update status to 'connected', reset failureCount
    - Return { synced: number of accounts synced/updated }
    - Log sync activity to activityLog

    **`markAccountDisconnected(accountId: number): void`**
    - Update socialAccount set status='disconnected' where id=accountId
    - Log to activityLog type: 'account', level: 'warn'

    **`incrementFailureCount(accountId: number): void`**
    - Increment failureCount for the account
    - If failureCount >= 5: call markAccountDisconnected
    - This function will be called by the publish cron in Phase 5, but we create it now for completeness

    **Create `src/app/(dashboard)/brands/[id]/accounts-section.tsx`.**
    Client component ('use client'):
    - Props: `{ brandId: number, accounts: SocialAccount[] }`

      - Username
      - Status badge: green "Connected" or red "Disconnected" (use shadcn Badge)
    - "Sync Accounts" button: calls syncAccounts server action, then refreshes page (router.refresh())
