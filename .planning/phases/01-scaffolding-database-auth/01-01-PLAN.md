    **Create `src/lib/circuit-breaker.ts`.**
    Use the exact pattern from 01-RESEARCH.md Pattern 5:
    - `CircuitBreaker` class with CLOSED/OPEN/HALF_OPEN states
    - Constructor takes service name and options (threshold default 5, cooldown default 300000ms)
    - `call<T>(fn: () => Promise<T>): Promise<T>` method that checks state, runs fn, handles success/failure transitions
    - On OPEN state: check cooldown elapsed, throw if still in cooldown with remaining seconds in message
    - On success: reset failures to 0, set state to CLOSED
    - On failure: increment failures, transition to OPEN if >= threshold or if HALF_OPEN
    - Export `getBreaker(service, opts?)` singleton registry using Map
    - Log circuit open events to console.error with service name and failure count

    **Create `src/lib/sanitize.ts`.**
    Use the pattern from 01-RESEARCH.md:
    - Export `sanitizeText(input: string): string`
    - Strip HTML tags: `/<[^>]*>/g` -> ''
    - Strip invisible Unicode: `/[\u200B-\u200D\uFEFF\u00AD\u2028\u2029]/g` -> ''
    - Normalize spaces/tabs: `/[ \t]+/g` -> ' '
    - Normalize excessive newlines: `/\n{3,}/g` -> '\n\n'
    - Trim

    **Create `src/lib/upload-post.ts`.**
    Use the pattern from 01-RESEARCH.md Pattern 7:
    - Base URL: `https://api.upload-post.com/api`
    - Auth header: `Authorization: Apikey {UPLOAD_POST_API_KEY}`
    - `listProfiles()`: GET /uploadposts/users, returns UploadPostProfile[]
    - `verifyApiKey()`: GET /uploadposts/me, returns account info
    - Type: `UploadPostProfile { username, connected_platforms: string[], status, created_at }`
    - Wrap calls with circuit breaker via `getBreaker('upload-post')`
    - Throw clear error if UPLOAD_POST_API_KEY not set

    **Create `src/lib/ai.ts`.**
    - Read `AI_MODE` env var (default 'testing')
    - Export `getModelConfig()` returning model names based on AI_MODE:
      - testing: { primary: 'claude-sonnet-4-20250514', critique: 'claude-haiku-3-20250307', filter: 'claude-haiku-3-20250307' }
      - production: { primary: 'claude-opus-4-20250514', critique: 'claude-sonnet-4-20250514', filter: 'claude-haiku-3-20250307' }
    - Export `checkAiSpend(): Promise<boolean>` — queries aiSpendLog for today's date, sums costUsd, returns true if under MAX_DAILY_AI_SPEND
    - Export `logAiSpend(params: { brandId?, model, inputTokens, outputTokens, costUsd }): void` — inserts row into aiSpendLog
    - Note: The actual Claude API call function will be created in Phase 2A when content generation is implemented. This file sets up the configuration and spend tracking.
    - Wrap API-calling functions with circuit breaker via `getBreaker('anthropic')`

    **Create `src/lib/r2.ts`.**
    Use the pattern from 01-RESEARCH.md Pattern 6:
    - `getR2Client()`: S3Client with endpoint `https://{R2_ACCOUNT_ID}.r2.cloudflarestorage.com`, region 'auto', credentials from env vars. Add `requestChecksumCalculation: 'WHEN_REQUIRED'` to handle SDK v3.729.0+ checksum issue.
    - `uploadToR2(bucket, key, body): Promise<void>`: PutObjectCommand
    - `listR2Objects(bucket, prefix): Promise<string[]>`: ListObjectsV2Command, returns sorted keys
    - `deleteR2Object(bucket, key): Promise<void>`: DeleteObjectCommand
    - `runDbBackup(): Promise<void>`: copies SQLite DB file to R2 with key `backups/app-{YYYY-MM-DD}.db`, then lists all backup keys, deletes oldest if > 7
    - Throw clear error if R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, or R2_SECRET_ACCESS_KEY not set
