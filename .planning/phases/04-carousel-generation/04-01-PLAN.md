function loadFonts() {
  const dir = path.join(process.cwd(), 'public', 'fonts')
  return [
    { name: 'Inter', data: fs.readFileSync(path.join(dir, 'Inter-Regular.woff')),  weight: 400, style: 'normal' as const },

3. **3 template functions** (all return Satori object-vnode, NOT JSX):

**`templateMinimal`**: Dark background (#0f0f0f), white text, brand primaryColor as accent for slide numbers and handle. CTA slide uses primaryColor as background. Uses justifyContent: 'space-between' for vertical spacing (NOT margin: 'auto'). Slide number top-left in accent color. Title 56px bold. Body 28px #cccccc.

**`templateBold`**: Full primaryColor background on hook + CTA slides, #ffffff background on content slides with dark text. Large title 64px bold. Energetic feel. Content slides have primaryColor left border strip (8px wide absolute-positioned div). Body text #333333 on light slides.

**`templateGradient`**: Top header bar (80px) with linear gradient from primaryColor to secondaryColor (or primaryColor at 60% opacity if no secondary). Rest of slide is #1a1a1a. Title below header bar 48px bold white. Body 26px #bbbbbb. CTA slide: full gradient background.

All 3 templates share:
- 1080x1080 dimensions (set in satori() call, not in template)
- Slide number indicator: "{slideNumber} / {totalSlides}" top-left, 16px, accent color
- Title rendered with fontWeight: 700
- Body rendered with fontWeight: 400
- CTA slide: ctaText in 32px fontWeight 600, handle in 20px accent color below CTA
- Logo on CTA slide: if logoUrl provided, render { type: 'img', props: { src: logoUrl, style: { width: 60, height: 60 } } } in bottom-right corner with position: 'absolute' (parent must be position: 'relative')
- Wrap logo img in try-catch pattern: if logo fails, slide renders without it (non-fatal, matching image-gen.ts watermark pattern)
- All children arrays use .filter(Boolean) to remove nulls
- CRITICAL: Use `flexDirection: 'column'` and `justifyContent: 'space-between'` on root container for CTA bottom-anchoring. Do NOT use `margin: 'auto'` for Satori compatibility.

4. **TEMPLATES registry:**
```typescript
const TEMPLATES: Record<TemplateId, (brand: BrandStyle, slide: SlideData) => object> = {
  minimal: templateMinimal,
  bold: templateBold,
  gradient: templateGradient,
}
```

5. **renderCarouselSlides function** (the main export):
```typescript
export async function renderCarouselSlides(params: {
  templateId: TemplateId
  brandId: number
  brandStyle: BrandStyle
  slides: SlideData[]
}): Promise<{ r2Keys: string[]; thumbKeys: string[] }>
```

Implementation:
- Load fonts once with loadFonts()
- Loop through slides sequentially:
  - Call TEMPLATES[templateId](brandStyle, slide) to get vnode
  - satori(vnode as any, { width: 1080, height: 1080, fonts }) -> SVG string
  - sharp(Buffer.from(svg)).png().toBuffer() -> PNG buffer
  - sharp(Buffer.from(svg)).resize(400, 400, { fit: 'cover' }).jpeg({ quality: 80 }).toBuffer() -> thumb buffer
- After all slides rendered, upload all to R2 in parallel using Promise.all:
  - R2 key pattern: `brands/${brandId}/carousels/${timestamp}/slide-${index}.png`
  - Thumb key pattern: `brands/${brandId}/carousels/${timestamp}/slide-${index}-thumb.jpg`
  - Use `uploadToR2(bucket, key, buffer)` from '@/lib/r2'
  - bucket = `process.env.R2_MEDIA_BUCKET ?? 'social-media'`
- Return { r2Keys, thumbKeys }

IMPORTANT anti-patterns from research:
- Do NOT use JSX syntax. Only object vnodes: { type: 'div', props: { style: {...}, children: [...] } }
- Do NOT import WOFF2 files (Satori rejects them)
- Do NOT use `gap` CSS property (Satori may not support it reliably) -- use margin/padding instead
- Do NOT load fonts per-slide -- load once per renderCarouselSlides call
- img elements MUST have both width AND height in style (Satori does not auto-size images)
