Implement multi-variant generation pipeline and brand toggle.

Purpose: When a brand enables multi-variant generation, the auto-generate pipeline produces 3 content variants at different Haiku temperatures, scores each with the existing Sonnet quality gate, and saves the highest-scoring variant as the canonical post while storing losers as linked draft records. This is the core MVAR feature — without it, no variant data exists.

Output: variant-generator.ts, modified auto-generate.ts with variant branching, brand-form toggle with cost estimate, updateBrand parsing enableVariants.

} from '@/app/actions/generate'

export interface VariantResult {

  qualityScore: number
  temperature: number
}

  winner: VariantResult
  losers: VariantResult[]
  variantGroup: string
  activeLearningIds: number[]
  totalCostUsd: number
  error?: string
}

The `generateVariants` function should:
1. Query brand from DB (same pattern as `generateContent`)
2. Load learnings + golden examples for `platforms[0]` (same as `generateContent` does)
3. Build system prompt and generation prompt (reuse exported helpers)
4. Call Haiku 3 times concurrently via `Promise.all`, each wrapped in `getBreaker('anthropic').call()`, using temperatures 0.7, 0.85, 1.0. Use `modelConfig.primary` (Haiku in testing mode) with `max_tokens: 4096`. Pass `temperature` parameter to `messages.create()`.
5. Log AI spend for each of the 3 generation calls individually via `logAiSpend()`
6. Parse each response with `parseJsonResponse<GeneratedContent>()`, extract platform contents
7. Run `runCritique(brandId, platforms[0], primaryContent, brand.voiceTone)` on each variant's primary platform content. Score on `platforms[0]` only per research Pitfall 1.
8. Sort by `overallScore` descending — index 0 is winner, rest are losers
9. Return `{ winner, losers, variantGroup: crypto.randomUUID(), activeLearningIds, totalCostUsd }`

If any individual Haiku call fails, log the error and continue with remaining variants. If ALL fail, return `{ error: 'All variant generation calls failed' }`. If fewer than 3 succeed, still pick the best among those that did succeed.
