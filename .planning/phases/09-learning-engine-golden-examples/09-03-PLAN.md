2. Query brand (name only for header).
3. Query all `brandLearnings` for this brand, ordered by `createdAt DESC`. Include all columns.
4. Count posts with non-null engagement scores for this brand (to check 30-post threshold): `SELECT count(*) FROM post_analytics INNER JOIN posts ON ... WHERE posts.brandId = ? AND post_analytics.engagement_score IS NOT NULL`.
5. Pass data to a client `LearningsSection` component.

**Client component (inline or same file using 'use client' section component pattern):**

Create a `LearningsSection` client component in the same file directory (e.g., `learnings-section.tsx`) that receives `learnings`, `brandId`, `hasEnoughData` (boolean: count >= 30), `dataCount` (actual count).

Layout:
- Back link: `<Link href={/brands/${brandId}}>` with ChevronLeft, same as analytics page.
- Title: `{brandName} Learnings`
- Subtitle: "AI-extracted patterns from your top and bottom performers"

**Run Analysis button area:**
- If `hasEnoughData` is true: Show a "Run Analysis" button that calls `runManualAnalysis(brandId)` via `useTransition`. Show loading state during the call.
- If `hasEnoughData` is false: Show disabled button with text "Not enough data yet -- need {30 - dataCount} more posts with engagement scores".

**Learnings table/list:**
- If no learnings exist: Show empty state card: "No learnings yet. Learnings are generated weekly from your top and bottom performing posts."
- If learnings exist, render as a list of cards (one per learning). Each card shows:

| Element | Content |
|---------|---------|
| Type badge | `hook_pattern` / `topic_pattern` / `structural_pattern` / `avoid_pattern`. For `avoid_pattern`, use a red/orange badge with "AVOID" prefix. Others use default badge. |
| Description | The learning description text |
| Confidence badge | "high" (green), "medium" (yellow/amber), "low" (grey) |
| Status indicator | See below |
| Supporting posts | "{N} supporting posts" where N = `supportingPostIds?.length ?? 0` |
| Platform | Platform badge if set, "All platforms" text if null |
| Created | Relative date (e.g., "3 days ago" or formatted date) |
| Actions | See below |

**Status indicators + actions by state:**
- `status=pending`: Amber/yellow background highlight. Actions: "Approve" (green button, calls `approveLearning`), "Reject" (red button, calls `rejectLearning`).
- `status=approved` + `isActive=1`: Green "Active" badge. Action: "Pause" button (calls `toggleLearning(id, false)`).
- `status=approved` + `isActive=0`: Grey "Paused" badge. Action: "Activate" button (calls `toggleLearning(id, true)`).
- `status=rejected`: Red "Rejected" badge. No actions. Collapsed/hidden by default (show a "Show rejected" toggle at the bottom).

Use `useTransition` for all server action calls with optimistic loading states.

Import server actions from `@/app/actions/learnings`.
