**Step 2: Create src/lib/learning-engine.ts**

Create the learning engine with these exports:
- `analyzeForBrand(brandId: number, platform: string, options?: { force?: boolean }): Promise<{ skipped?: boolean; reason?: string; learningsWritten?: number }>`
- `analyzeAllPlatformsForBrand(brandId: number, options?: { force?: boolean }): Promise<void>`
- `analyzeAllBrands(): Promise<void>`

Implementation details for `analyzeForBrand()`:
1. **Time gate:** Query `brands.lastLearningRunAt`; if ran within 7 days AND `force` is not true, return `{ skipped: true, reason: 'ran_recently' }`.
2. **Data gate:** Count `postAnalytics` rows joined to `posts` where `posts.brandId = brandId`, `postAnalytics.platform = platform`, AND `postAnalytics.engagementScore IS NOT NULL`. If < 30, return `{ skipped: true, reason: 'insufficient_data' }`. CRITICAL: filter by `isNotNull(postAnalytics.engagementScore)` -- not just row count.
3. **Spend gate:** Call `checkAiSpend()` from ai.ts. If false, return `{ skipped: true, reason: 'spend_limit' }`.
4. **Query top 20 and bottom 10:** Select `posts.id`, `posts.content`, `postAnalytics.engagementScore`, `postAnalytics.platform` from posts + postAnalytics joined, filtered by brandId + platform + `isNotNull(engagementScore)`, ordered by `engagementScore DESC`. Slice top 20 and bottom 10.
5. **Format for analysis:** Use `formatPostForAnalysis()` helper that returns: `POST ID: {id}\nENGAGEMENT SCORE: {score}\nHOOK (first 80 chars): {content.slice(0, 80)}\nLENGTH: {content.length} chars\nPLATFORM: {platform}`. Do NOT send full post content (token budget + security).
6. **Claude analysis call:** Use lazy-initialized Anthropic client (same pattern as generate.ts). Model: `getModelConfig().primary` (Sonnet). System prompt instructs extraction of max 5 learnings as JSON array: `[{ type: "hook_pattern"|"topic_pattern"|"structural_pattern"|"avoid_pattern", description: "max 100 chars", confidence: "high"|"medium"|"low", supportingPostIds: [1,2] }]`. User prompt sends top performers + underperformers.
7. **Parse response:** Strip markdown fences and JSON.parse. Use same pattern as `parseJsonResponse` in generate.ts (copy the logic, do NOT import from generate.ts since it's a server action file).
8. **Write learnings:** Insert each learning into `brandLearnings` with `brandId`, `platform`, `type`, `description`, `confidence`, `supportingPostIds`, `isActive: 1`, `status: 'pending'`, current ISO timestamps for createdAt/updatedAt. Batch all inserts.
9. **Update lastLearningRunAt:** Set `brands.lastLearningRunAt = now` for this brandId.
10. **Log spend:** Call `logAiSpend()` with brandId, model, tokens, cost. Use `calculateCostUsd` logic inline (copy the pricing table, do NOT import from generate.ts server action).
11. **Mutex:** Use `globalThis.__learningRunning` pattern (same as `__analyticsRunning` in collect-analytics.ts) to prevent concurrent runs.

`analyzeAllPlatformsForBrand(brandId, options?)`: Query distinct platforms from `postAnalytics` joined to `posts` for this brand. Call `analyzeForBrand()` for each platform sequentially.

`analyzeAllBrands()`: Query all brand IDs. Call `analyzeAllPlatformsForBrand()` for each. Wrap in mutex guard.

**Step 3: Create src/lib/prompt-injector.ts**

Create with two exports:

`loadLearnings(brandId: number, platform: string): BrandLearning[]`
- Query `brandLearnings` WHERE `brandId`, `isActive = 1`, `status = 'approved'`, AND (`platform IS NULL` OR `platform = platform`).
- Order by confidence (CASE: high=0, medium=1, low=2) then `validatedAt DESC`.
- LIMIT 5.
- CRITICAL: Both `isActive = 1` AND `status = 'approved'` must be present. `isActive` alone is NOT sufficient (Pitfall 4 from research).

`loadGoldenExamples(brandId: number, platform: string): GoldenExample[]`
- Query all published posts with engagement scores for brand+platform, excluding `variantOf IS NOT NULL` losers.
- Calculate 90th percentile threshold from sorted scores: `scores[Math.floor(scores.length * 0.90)]`.
- Split into three groups: pinned (`isGoldenPinned = 1`), recent top (last 30 days, score >= p90), historic top (older, score >= p90).
- Return `[...pinned, ...recentTop, ...historicTop].slice(0, 5)`.
- Each result includes: `postId`, `content`, `engagementScore`, `platform`, `collectedAt`, `isGoldenPinned`.

Export `BrandLearning` and `GoldenExample` types for use by Plan 02.

**Step 2: Add threshold-gated learning trigger to collectAnalytics()**

In `collect-analytics.ts`, after the reclassification loop (the `for (const cohortKey of affectedCohorts)` block), add a learning analysis trigger:

1. Track how many newly reclassified top+under posts exist per brand during reclassification. Use the existing `affectedCohorts` set. For each cohort key, count how many rows changed tier to 'top' or 'under' during this run. The simplest approach: after the reclassification loop completes, compute per-brand counts from `affectedCohorts` (which tracks "brandId:platform" keys).

2. Group by brandId (one brand may have multiple platform cohorts). If a brand has 5+ newly reclassified cohort entries (summed across platforms), trigger learning analysis:

```typescript
// After reclassification loop, before the log summary:
const brandCohortCounts = new Map<number, number>()
for (const cohortKey of affectedCohorts) {
  const brandId = parseInt(cohortKey.split(':')[0], 10)
  brandCohortCounts.set(brandId, (brandCohortCounts.get(brandId) ?? 0) + 1)
}

for (const [brandId, count] of brandCohortCounts) {
  if (count >= 2) { // 2+ platform cohorts were affected = meaningful new data
    try {
      const { analyzeAllPlatformsForBrand } = await import('./learning-engine')
      await analyzeAllPlatformsForBrand(brandId)
    } catch (err) {
      console.error(`[collect-analytics] learning trigger failed for brand ${brandId}:`, err)
    }
  }
}
```

Note: `analyzeForBrand()` internally checks the 7-day time gate and 30-post minimum, so calling it is safe even if conditions aren't met -- it returns early with `{ skipped: true }`.
