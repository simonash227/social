<task type="auto">
  <name>Task 2: Build collect-analytics cron worker + register in cron.ts</name>
  <files>src/lib/collect-analytics.ts, src/lib/cron.ts</files>
  <action>
Create `src/lib/collect-analytics.ts` with:

1. **fetchPostAnalytics(requestId: string)** -- internal helper:
   - GET `https://api.upload-post.com/api/uploadposts/post-analytics/${requestId}`
   - Headers: `{ Authorization: 'Apikey ' + process.env.UPLOAD_POST_API_KEY }`
   - Returns parsed JSON or null on non-200 (non-fatal: log warn, continue)
   - The response contains per-platform entries with: `{ success, platform_post_id, post_url, post_metrics: { views, likes, comments, shares }, post_metrics_source, post_metrics_error? }`
   - If `post_metrics_error` is present, skip that platform's metrics (log warn)

2. **calcEngagementScore(metrics: { views: number | null, likes: number | null, comments: number | null, shares: number | null }): number** -- exported:
   - Guard: if views is null or 0, return 0 (ANLY-02 explicit requirement)
   - Weighted engagement: (likes * 1) + (comments * 3) + (shares * 2)
   - Score = Math.min(100, Math.round((weighted / views) * 1000))
   - Use nullish coalescing to default each metric to 0

3. **classifyTier(score: number, p25: number, p75: number): 'top' | 'average' | 'under'** -- exported:
   - score > p75 -> 'top'
   - score < p25 -> 'under'
   - else -> 'average'

4. **collectAnalytics(): Promise<void>** -- exported, main entry:
   - Mutex guard: `if (globalThis.__analyticsRunning) return` (same pattern as publishDuePosts)
   - Set `globalThis.__analyticsRunning = true` in try/finally
   - Find eligible postPlatforms: JOIN posts with postPlatforms WHERE posts.status = 'published' AND posts.publishedAt is not null AND posts.publishedAt <= 48 hours ago AND postPlatforms.status = 'published' AND postPlatforms.requestId IS NOT NULL
   - For each eligible postPlatform with requestId:
     - Call fetchPostAnalytics(requestId)
     - For each platform result in the response:
       - If post_metrics_error exists, log warn to activityLog and skip
       - Compute engagementScore via calcEngagementScore
       - Upsert into postAnalytics (INSERT ... ON CONFLICT(postId, platform) DO UPDATE) -- Note: add a unique index/constraint on (postId, platform) for the upsert to work. If drizzle doesn't support upsert cleanly, use a SELECT first + INSERT or UPDATE pattern.
     - Log success to activityLog (type: 'analytics', level: 'info')
   - After all collection done, reclassify tiers:
     - Group all postAnalytics rows by (brandId, platform) -- join through posts table to get brandId
     - For each cohort: if fewer than 4 posts, set all to 'average'. Otherwise sort engagement scores, compute p25 = scores[Math.floor(len * 0.25)], p75 = scores[Math.floor(len * 0.75)], update performerTier for each row
     - Skip cohort entries where engagementScore is 0 (impressions=0) from percentile calculation, but still classify them as 'under'
   - Log summary to activityLog: "Collected analytics for N posts, reclassified M cohorts"

Add to `src/lib/cron.ts`:
- Register cron schedule `'0 */6 * * *'` (every 6 hours) in the `initCron` function
- Use dynamic import pattern: `const { collectAnalytics } = await import('./collect-analytics')`
- Wrap in try/catch, log error to console.error on failure
- Update the console.log at the bottom to include 'collect-analytics' in the registered jobs list
