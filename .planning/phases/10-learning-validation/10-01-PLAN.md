Implement `deriveConfidence(postCount: number, deltaMagnitude: number)`:
- `postCount >= 20 && deltaMagnitude >= 5` -> 'high'
- `postCount >= 10 && deltaMagnitude >= 2` -> 'medium'
- Otherwise -> 'low'

Implement `computeLearningStats(brandId: number, learningId: number): LearningStats`:
- Use `getDb()` from `@/db` (synchronous better-sqlite3).
- Query "with" group using raw SQL via `sql` tagged template: `SELECT pa.engagement_score AS "engagementScore" FROM post_analytics pa INNER JOIN posts p ON pa.post_id = p.id INNER JOIN json_each(pa.active_learning_ids) jl ON jl.value = ${learningId} WHERE p.brand_id = ${brandId} AND pa.engagement_score IS NOT NULL`
- Query "without" group: same join but `WHERE ... AND (pa.active_learning_ids IS NULL OR NOT EXISTS (SELECT 1 FROM json_each(pa.active_learning_ids) jl2 WHERE jl2.value = ${learningId}))`
- Use `db.all<{ engagementScore: number }>(sql`...`)` for both queries.
- Calculate averages, delta, and confidence. Round averages and delta with `Math.round()`.
- Return the LearningStats object.

Implement `autoDeactivateLearnings(options?: { threshold?: number }): number`:
- Default threshold = 20.
- Query all learnings with `isActive=1 AND status='approved'` from `brandLearnings`.
- For each, call `computeLearningStats()`.
- If `stats.withCount >= threshold && stats.engagementDelta <= 0`, update the learning: set `isActive: 0`, `status: 'auto_deactivated'`, `updatedAt: new Date().toISOString()`.
- Log each deactivation with `console.log`.
- Return count of deactivated learnings.

Note: Keep `computeLearningStats` synchronous (better-sqlite3 is sync). The cron wrapper can be async.

```typescript
export async function getLearningEffectiveness(brandId: number): Promise<Record<number, LearningStats>> {
  const db = getDb()
  const { computeLearningStats } = await import('@/lib/learning-validator')

  const learnings = db.select({ id: brandLearnings.id, status: brandLearnings.status })
    .from(brandLearnings)
    .where(eq(brandLearnings.brandId, brandId))
    .all()

  const result: Record<number, LearningStats> = {}
  for (const l of learnings) {
    if (l.status === 'approved' || l.status === 'auto_deactivated') {
      result[l.id] = computeLearningStats(brandId, l.id)
    }
  }
  return result
}
```

Import `LearningStats` type from `@/lib/learning-validator`. Use dynamic import for the function to keep server action boundary lightweight.

**IMPORTANT:** `autoDeactivateLearnings` is NOT async (better-sqlite3 is sync). The cron wrapper wraps it in async for error handling, but the function itself returns `number` not `Promise<number>`.
