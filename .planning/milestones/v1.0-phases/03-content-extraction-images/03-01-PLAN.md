<task type="auto">
  <name>Task 1: Install extraction dependencies and create content extraction module</name>
  <files>package.json, next.config.ts, src/lib/extract.ts</files>
  <action>
1. Install dependencies:
   ```
   npm install youtube-transcript @mozilla/readability jsdom pdf-parse
   npm install --save-dev @types/jsdom @types/pdf-parse
   ```

2. Add `pdf-parse` to `serverExternalPackages` in `next.config.ts` alongside `better-sqlite3` and `node-cron`. This prevents Next.js standalone bundler from breaking on pdf-parse's dynamic require() of test files (see research pitfall 5).

3. Create `src/lib/extract.ts` with two exported functions:

   **`extractFromUrl(url: string): Promise<{ text: string; title?: string; error?: string }>`**
   - Detect YouTube URLs via regex: `/(?:youtu\.be\/|youtube\.com\/watch\?v=)([a-zA-Z0-9_-]{11})/`
   - For YouTube: call `YoutubeTranscript.fetchTranscript(url)`, join segment `.text` with spaces. Wrap in try/catch -- return `{ text: '', error: 'YouTube transcript unavailable...' }` on failure.
   - For non-YouTube URLs: `fetch(url, { headers: { 'User-Agent': 'Mozilla/5.0 ...' } })`, get HTML text, create `new JSDOM(html, { url })`, run `new Readability(dom.window.document).parse()`. Return `{ text: article.textContent.trim(), title: article.title }`. On failure return `{ text: '', error: 'Could not extract article content' }`.
   - Add a 5MB content length guard: if HTML response body > 5MB, truncate before passing to JSDOM (prevents memory spikes per research pitfall 2).
   - Never throw -- always return an error string in the result object.

   **`extractPdf(buffer: Buffer): Promise<{ text: string; error?: string }>`**
   - Call `pdfParse(buffer)`, return `{ text: data.text.trim() }`.
   - Wrap in try/catch, return `{ text: '', error: 'Could not extract PDF content' }` on failure.
   - Never throw.
  </action>

<task type="auto">
  <name>Task 2: Integrate extraction into generate action and add extractSource server action</name>
  <files>src/app/actions/generate.ts</files>
  <action>
1. Import `extractFromUrl` and `extractPdf` from `@/lib/extract` at the top of the file.

2. Add a new exported server action `extractSource`:
   ```typescript
   export async function extractSource(
     sourceUrl: string,
     pdfBase64?: string
   ): Promise<{ text: string; title?: string; error?: string }>
   ```
   - If `pdfBase64` is provided: decode from base64 to Buffer, call `extractPdf(buffer)`, return result.
   - Else if `sourceUrl` is provided: call `extractFromUrl(sourceUrl)`, return result.
   - Else: return `{ text: '' }`.
   - This action is called from the client BEFORE generateContent, so the extracted text is visible to the user and editable before generation.

3. Modify the `generateContent` function:
   - After sanitizing sourceText (step 3), if `sourceText` is empty but `sourceUrl` is provided, call `extractFromUrl(sourceUrl)` to auto-extract. Set `cleanSourceText` to the extracted text. This ensures that even if the user skips the explicit extraction step, content gets extracted.
   - If extraction returns an error and no text, continue with existing URL-only prompt behavior (graceful degradation -- do NOT fail the generation).

4. Keep all existing exports and function signatures unchanged. The `extractSource` action is additive.
  </action>

  </verify>
