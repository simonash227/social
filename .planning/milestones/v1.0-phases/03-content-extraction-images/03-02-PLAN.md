<objective>
Build the image generation backend: OpenAI gpt-image-1 integration, sharp watermarking, R2 storage with thumbnails, and database tracking.

Purpose: This plan creates the complete backend pipeline for AI image generation with brand consistency. When a user triggers image generation, the system calls gpt-image-1 with brand style directives, applies the brand's logo as a watermark, generates a 400px thumbnail, uploads both to R2, and records the metadata. This backend is consumed by Plan 03's UI.

Output: `generatedImages` DB table, `src/lib/image-gen.ts` module, `src/app/actions/images.ts` server actions, updated `src/lib/r2.ts` with public URL helper.
</objective>

   ```typescript
   /**
    * Get the public URL for an R2 object in the media bucket.
    * Requires R2_MEDIA_PUBLIC_BASE env var (e.g. https://media.example.com).
    */
   export function getR2PublicUrl(key: string): string {
     const base = process.env.R2_MEDIA_PUBLIC_BASE
     if (!base) throw new Error('R2_MEDIA_PUBLIC_BASE env var not set')
     return `${base.replace(/\/$/, '')}/${key}`
   }
   ```

1. Create `src/lib/image-gen.ts`:

   **Module-level OpenAI client** (follows same pattern as Anthropic client in generate.ts):
   ```typescript
   import OpenAI from 'openai'
   const openai = new OpenAI() // reads OPENAI_API_KEY from env
   ```

   **Export `generateBrandImage(params)`** with this signature:
   ```typescript
   export async function generateBrandImage(params: {
     prompt: string
     brandStyleDirective: string
     logoUrl?: string
     watermarkPosition?: 'top-left' | 'top-right' | 'bottom-left' | 'bottom-right'
     watermarkOpacity?: number
     brandId: number
   }): Promise<{ fullKey: string; thumbKey: string; costUsd: string }>
   ```

   Implementation:
   a. Call `openai.images.generate()` with:
      - `model: 'gpt-image-1'`
      - `prompt: params.brandStyleDirective + '\n\n' + params.prompt`
      - `size: '1024x1024'`
      - `quality: 'medium'`
   b. Extract base64 from `response.data[0].b64_json!` (NOT `.url` -- gpt-image-1 only returns b64_json per research pitfall 6).
   c. Convert to Buffer: `Buffer.from(base64, 'base64')`
   d. If `params.logoUrl` is provided, call internal `applyWatermark()`:
      - Fetch logo from URL, convert to Buffer
      - Resize logo to 150px width via `sharp(logoBuffer).resize(150).png().toBuffer()`
      - Use `sharp(imageBuffer).composite([{ input: resizedLogo, gravity: GRAVITY_MAP[position] }]).png().toBuffer()`
      - GRAVITY_MAP: 'top-left'='northwest', 'top-right'='northeast', 'bottom-left'='southwest', 'bottom-right'='southeast'
      - For opacity: if watermarkOpacity is provided and sharp 0.33+ supports it, use the composite opacity parameter. Otherwise, skip advanced opacity handling for now (acceptable -- can refine later).
   e. Generate 400px thumbnail: `sharp(imageBuffer).resize(400, 400, { fit: 'cover', position: 'center' }).jpeg({ quality: 80 }).toBuffer()`
   f. Upload both to R2:
      - `const bucket = process.env.R2_MEDIA_BUCKET ?? 'social-media'`
      - `const timestamp = Date.now()`
      - Full key: `brands/${params.brandId}/images/${timestamp}.png`
      - Thumb key: `brands/${params.brandId}/images/${timestamp}-thumb.jpg`
      - Call `uploadToR2(bucket, fullKey, imageBuffer)` and `uploadToR2(bucket, thumbKey, thumbBuffer)`
   g. Return `{ fullKey, thumbKey, costUsd: '0.042' }` (medium quality 1024x1024 pricing).

   **Internal `applyWatermark()` function** -- not exported, used by generateBrandImage.
